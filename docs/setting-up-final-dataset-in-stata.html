Final Dataset in Stata

Learning outcomes

By the end of this module, you should be able to:

  1. Describe the full Stata cleaning workflow from GGIR outputs to a final .dta.
  2. Merge person-level, day-level, and night-level files without losing the unit of observation.
  3. Explain why Spearman-Brown Prophecy reliability is useful for accelerometer outcomes.
  4. Save a dated final dataset with reproducible exclusion rules.

Lesson 7.1

Final-Dataset Workflow

The course cleaning workflow can be written as a numbered Stata plan:

  1. Import part 5 files: import part5_personsummary_WW_..._T... files and preserve the original file names.
  2. Clean ID variables: standardize participant IDs, remove spaces or formatting artifacts, and confirm uniqueness.
  3. Append each lot: append each collection lot/batch into a single person-level dataset while keeping a batch indicator.
  4. Merge person + day files: merge person-level summaries with part2_daysummary or other day-level outputs using the approved ID/date structure.
  5. Clean sleep records: merge or process part4_nightsummary, then drop sleep values coded 2-6 when following the Pelotas/UFPel example rule.
  6. Generate MVPA bouts: create 1-minute, 5-minute, and 10-minute MVPA bout variables plus 20-minute inactivity indicators if GGIR did not automatically create the exact variables needed.
  7. Generate n-days variables: count valid days, weekday/weekend days, and records meeting ≥10 hours awake wear.
  8. Evaluate reliability: apply the Spearman-Brown Prophecy method to estimate how reliability changes with the number of days.
  9. Collapse to wide weekly means: convert from day-level records to participant-level weekly averages or wide-format variables required by the analysis.
  10. Apply three-parameter exclusions: exclude records that fail the approved combination of valid-day count, awake wear duration, and weekend-day requirement.
  11. Save a dated .dta: save the final dataset with a date stamp and export a data dictionary/log.

示例: Pelotas/UFPel cohort
The UFPel do-file workflow should be credited to Otavio Leão, Giulia Salaberry, and Charles Lucena. Treat it as a strong reproducible example, not a universal template: your study must decide its own ID cleaning, valid-day rules, bout definitions, and exclusion criteria.

Lesson 7.2

Spearman-Brown Prophecy Reliability

The Spearman-Brown Prophecy method estimates how reliability changes when the number of measurement days changes. In accelerometer studies, this is useful because a single day may be noisy, while multiple valid days better approximate habitual activity.

The transferable idea is:

  1. estimate reliability for the current number of valid days,
  2. project reliability if more or fewer days are used,
  3. use the result to justify minimum valid-day rules,
  4. report the rule transparently in the methods section.

This is a valuable method to teach because it connects data cleaning with measurement quality rather than treating valid days as an arbitrary cutoff.

Lesson 7.3

Three-Parameter Exclusion Rule

For the course example, final inclusion is evaluated using three parameters:

Parameter Example rule
Number of valid days ≥4 valid days
Awake wear duration ≥10 hours awake wear per valid day
Weekend coverage At least 1 weekend day

Keep the exclusion log. It should report how many participants were excluded by each rule, how many failed multiple rules, and the final analytic sample.

Critical review point: Do not define validity rules, aggregation procedures, or outcome thresholds only after seeing results. The data-management and analysis plans should guide these decisions.

Lesson 7.4

Save Final Outputs

Final outputs should include:

  1. dated .dta file,
  2. data dictionary,
  3. cleaning log,
  4. exclusion table,
  5. Stata do-file,
  6. README with source GGIR run and batch dates.

Example file name:

accelerometer_final_analysis_YYYYMMDD.dta

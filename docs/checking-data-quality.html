Module 5

Checking Data Quality

Use logs, summaries, plots, and protocol rules to decide whether processed accelerometer files are trustworthy enough to clean and analyze.

QC outputs Calibration and clipping Valid-day review

Learning outcomes

By the end of this module, you should be able to:

  1. Locate the main GGIR QC outputs.
  2. Explain clipping_score, calib_err, and complete_24hcycle.
  3. Distinguish QC visual inspection files from participant-facing reports.
  4. Document quality decisions before final cleaning.
5.1 Locate QC outputs 5.2 Read QC variables 5.3 Visual inspection 5.4 Valid-day review

Lesson 5.1

Locate Real QC Outputs

After GGIR processing, inspect the output folder before opening Stata. The exact folder structure can vary by GGIR version and options, but the following outputs should be explicitly checked when present:

Output What to look for
results/QC/data_quality_report.csv Summary indicators for calibration, clipping, missingness, and recording quality.
Per-file .png quality plots Visual patterns that reveal odd days, missing segments, non-wear blocks, or device problems.
PDF visualizations Higher-level plots useful for review, communication, or documentation.
Time_report_... files and related .png plots QC-oriented visual inspection of the wear pattern and daily signal.

Important correction: The visualreport produced around part 5 is mainly participant-facing / communication-oriented. For QC, rely on Time_report_... files, .png quality plots, and QC summaries rather than treating the part 5 visual report as the primary QC tool.

Lesson 5.2

Read the QC Variables

Three variables deserve special attention because they often appear in quizzes or downstream filters:

Variable Meaning How to use it
clipping_score Indicator of whether the acceleration signal hits the sensor range, suggesting signal saturation or clipping. High or unusual clipping requires review before trusting intensity summaries.
calib_err Calibration error, often interpreted in mg (milligravity units). Large calibration error can indicate poor calibration or device/signal problems; compare to the study’s approved threshold.
complete_24hcycle Whether a recording/day contains a complete 24-hour cycle according to the processing definition. Useful for identifying days with complete day-night coverage, especially in 24-hour protocols.

These variables should be taught before they appear in Module 8. Learners should know both the variable name and the practical decision it supports.

Lesson 5.3

Visual Inspection

Numerical summaries can miss problems that are obvious in plots. During visual inspection, look for:

  1. flat lines suggesting non-wear, device malfunction, or removal,
  2. abrupt shifts that suggest device handling or placement change,
  3. unexpectedly short or long recordings,
  4. missing day-night cycles,
  5. calibration or clipping warnings,
  6. disagreement between logs and processed output.

Visual QC should be reproducible. Keep a QC log with:

  • participant ID,
  • file name,
  • date reviewed,
  • reviewer,
  • concern category,
  • action taken,
  • whether a second reviewer is needed.

Lesson 5.4

Valid-Day Review

A non-wear interval is a period when the device is likely not being worn. A valid day is a day that meets the study’s minimum usable-data criteria. For the course example, valid-day review uses:

  1. at least 4 valid days,
  2. at least 10 hours of awake wear per valid day,
  3. at least 1 weekend day.

Do not apply these rules silently in Stata. First confirm the GGIR outputs contain the variables needed to evaluate them and document any exceptions.

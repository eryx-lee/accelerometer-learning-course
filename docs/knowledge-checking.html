Self-check

Quick Interactive Checks

Question 1: GGIR processing in this course primarily uses which ActiGraph file type?

Answer: .gt3x raw file.

Question 2: A 5-second .agd file is best described as what?

Answer: A count-based epoch summary.

Question 3: ENMO stands for what?

Answer: Euclidean Norm Minus One.

Question 4: MVPA means what?

Answer: Moderate-to-vigorous physical activity.

Question 5: Which current GGIR parameter name should be taught?

Answer: data_masking_strategy.

Question 6: The GGIR documentation lists six numbered parts. True or false?

Answer: True.

Question 7: In the course example, a valid participant record requires at least 4 valid days, ≥10 hours awake wear, and at least 1 weekend day. True or false?

Answer: True.

Question 8: The part 5 visualreport should be treated as the primary QC tool. True or false?

Answer: False. It is mainly participant-facing / communication-oriented; QC should use Time_report_... files, .png plots, and QC summaries when present.

Question 9: The QC variable calib_err is a calibration error commonly interpreted in mg. True or false?

Answer: True.

Question 10: An incorrect consumer-device row with a non-real product name should be retained in the device table. True or false?

Answer: False. Incorrect device labels should be removed or corrected rather than taught.

Application

Applied Questions

Scenario 1: A participant was scheduled for 7 days of recording, but the returned monitor contains less than 3 days of usable collection time. Which first-pass folder should it go into?

Answer: INCOMPLETE.

Less than half of the planned collection period should be triaged as INCOMPLETE. This is an operational folder rule; final valid-day rules are evaluated later after processing.

Scenario 2: In part5_personsummary_WW_L30M200V700_T5A3.csv, what does M200 represent?

Answer: Moderate-intensity threshold/cut-point = 200.

L30M200V700 encodes light, moderate, and vigorous cut-points. T5 and A3 encode the time and angle thresholds.

Scenario 3: You want to inspect calibration, clipping, missingness, and recording-quality summaries. Which file is the best starting point when present?

Answer: results/QC/data_quality_report.csv.

Scenario 4: In the Pelotas/UFPel example, sleep values coded 2-6 are dropped during cleaning. Is this a universal rule for every accelerometer study?

Answer: No. It is an example rule that must be verified for each study.

Scenario 5: Why is the Spearman-Brown Prophecy method useful in this course?

Answer: It estimates how reliability changes with the number of measurement days.

Review

Short-Answer Review

Write short answers to these prompts, then compare with the suggested answers.

  1. What is the difference between a raw signal and an activity outcome?
  2. Why does placement affect interpretation?
  3. Why should conversion from .gt3x to .csv be described as pipeline-dependent?
  4. What is the difference between a participant-level file and a day-level file?
  5. Why should exclusion rules be decided before examining study associations?
Suggested answers
  1. A raw signal is the sensor’s recorded acceleration time series; an outcome is a processed estimate derived using documented methods.
  2. Placement changes what motion is captured and determines which algorithms, thresholds, and interpretations are appropriate.
  3. GGIR can read .gt3x directly, so CSV conversion is needed only if a downstream pipeline requires it.
  4. Participant-level files summarize one person; day-level files contain separate records for each participant-day.
  5. Pre-specified rules reduce bias and make the analytic sample defensible.

Final reflection

Before Using Real Study Data

You should be able to explain:

  • which file type is used at each stage,
  • where QC outputs live,
  • what the main GGIR parameters mean,
  • how valid days and sleep records are cleaned,
  • how the final Stata dataset is assembled,
  • where each decision is documented.

Welcome

This course teaches the practical workflow for accelerometer-based research: planning data collection, programming and downloading devices, organizing files, processing raw data, checking quality, cleaning outputs, and creating an analysis-ready dataset.

The course uses ActiGraph examples, GGIR in R, and Stata as a reproducible workflow. The principles are broader than any one device or software package, and every study should document its own protocol and analytical decisions.

0.1 How to use this course

Each module includes short lectures, a video placeholder for the teaching recording, an optional practical activity, and key review points. Follow the modules in order for a first pass; return to the software demonstrations as you apply the workflow to a study.

0.4 Before you begin

You will need access to a research-grade accelerometer and its device-management software. Later modules use R/RStudio with GGIR and Stata (or an equivalent statistical package).

Important: Device configuration, valid-day rules, wear-time criteria, thresholds, and processing arguments must be selected for the study population, device, wear location, and protocol. This course explains the decision process; it does not prescribe one universal parameter set.

0.5 What you will need

  • A research-grade accelerometer and its approved device-management software.
  • Access to the study protocol and a file-storage location approved by the research team.
  • R/RStudio with GGIR for the processing modules.
  • Stata or an equivalent statistical package for the final-dataset module.

0.6 Course workflow

Plan and deploy → download → organize → process → inspect quality → clean and standardize → prepare the final dataset → document and report.
